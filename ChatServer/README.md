# ChatServer

이 프로젝트는 MFC 기반의 TCP 채팅 서버 예제입니다. 클라이언트가 접속하면 연결을 관리하고, 수신한 메시지를 화면에 표시한 뒤 현재 연결된 모든 클라이언트에게 브로드캐스트하는 구조로 되어 있습니다.

## 1. 한 줄 요약
- 서버는 특정 포트(21000)에서 클라이언트의 접속을 기다립니다.
- 클라이언트가 접속하면 별도 소켓 객체를 생성해 관리합니다.
- 메시지를 받으면 UI에 출력하고, 같은 메시지를 다른 클라이언트들에게도 전달합니다.

## 2. 실행 흐름
1. 서버 실행
2. ChatServerDlg의 OnInitDialog()에서 ListenSocket을 생성하고 `Listen()` 합니다.
3. 클라이언트가 접속하면 CListenSocket의 OnAccept()가 호출됩니다.
4. 각 클라이언트는 CClientSocket 객체로 관리됩니다.
5. 클라이언트가 메시지를 보내면 CClientSocket::OnReceive()가 처리합니다.
6. 서버는 받은 메시지를 화면에 보이고, 모든 클라이언트에게 다시 전송합니다.

## 3. 통신 구조
```text
클라이언트 연결 요청
   ↓
CListenSocket::OnAccept()
   ↓
새로운 CClientSocket 생성
   ↓
클라이언트 메시지 수신
   ↓
CClientSocket::OnReceive()
   ↓
서버 UI에 표시
   ↓
CListenSocket::SendChatDataAll()
   ↓
다른 클라이언트들에게 브로드캐스트
```

## 4. 주요 역할
### ChatServerDlg
- 서버 윈도우(UI) 담당
- 서버 시작 시 리스닝 소켓을 초기화합니다.
- 프로그램 종료 시 연결된 클라이언트 소켓들을 정리합니다.

### CListenSocket
- 서버의 메인 수신 소켓 역할을 합니다.
- 클라이언트 접속을 수락하고, 연결된 클라이언트 목록을 관리합니다.
- 브로드캐스트 전송을 담당합니다.

### CClientSocket
- 각각의 클라이언트와 1:1로 연결되는 소켓 객체입니다.
- 해당 클라이언트가 보낸 데이터를 수신합니다.
- 연결이 끊기면 정리 작업을 수행합니다.

## 5. 데이터 흐름
- 클라이언트가 메시지를 전송하면 서버의 `OnReceive()`가 이를 받습니다.
- 서버는 수신한 문자열을 `[%IP:%Port%]: 메시지` 형태로 가공합니다.
- 그 내용을 서버 UI 리스트 박스에 추가합니다.
- 마지막으로 `SendChatDataAll()`을 호출해 모든 연결된 클라이언트에게 동일한 메시지를 전달합니다.

## 6. 참고 포인트
- 현재 예제는 로컬 테스트용으로 127.0.0.1:21000을 사용합니다.
- 실제 서비스 수준에서는 인증, 메시지 포맷, 방 관리, 멀티스레딩 등 추가 설계가 필요합니다.
- 클라이언트가 종료하면 서버 쪽에서 해당 소켓을 정리합니다.
